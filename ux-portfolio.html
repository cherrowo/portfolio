✦ how I work

the process underneath

According to my User Design Methods Professor, I'm a 'Mixed Methods Researcher'. I use whatever gets me to a clear picture of what's wrong and a defensible answer for what to build instead.

🔍
01 · discover

Find the actual problem

User interviews, card sorts, analytics audits, competitive analysis. I don't start designing until I understand the problem and positioning.

🗺️
02 · structure

Build the skeleton first

Information architecture before visuals. I map user flows, define mental models, and work out the logic first. This is where most design problems get solved.

✏️
03 · prototype

Make something testable fast

Low-fidelity first, high-fidelity only once the structure holds. I prototype at the fidelity that answers the question I'm trying to answer, which is how I ship fast.

🧪
04 · test

Put it in front of real people

Usability testing, A/B experiments where applicable, qualitative synthesis.

📊
05 · measure

Define success before you ship

I define metrics before building, not after. Whether it's feature adoption, task completion rate, or complaint volume.

🔄
06 · iterate

Ship, then keep going

Launch isn't the end. I document what I shipped, and continue to improve.
